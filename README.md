# Рекомендательная система для оперативной корректировки расписания и управления аудиторным фондом НИТУ МИСИС

Веб-приложение для оперативного управления расписанием и аудиторным фондом. Позволяет массово переносить занятия при инцидентах (закрытие аудиторий/корпусов), бронировать аудитории под мероприятия и анализировать использование фонда.

## Возможности

### Инциденты — массовый перенос занятий

При закрытии аудитории или корпуса система автоматически подбирает оптимальные замены:

- Указываем тип инцидента (закрытие аудитории / корпуса / конкретных аудиторий)
- Указываем период (диапазон дат)
- Система находит все затронутые занятия и с помощью **венгерского алгоритма** назначает оптимальные замены
- Учитываются: вместимость, оборудование (ПК, проектор), принадлежность корпусу
- Потоковые лекции (несколько групп одновременно) переносятся целиком — все группы в одну аудиторию
- Результат: таблица назначений с оценкой качества (штраф, % пригодности)
- Неудачные назначения (не хватило аудиторий) показываются отдельно

### Бронирование мероприятий

Поиск свободных аудиторий для разовых мероприятий:

- Указываем параметры: дата, время, количество участников, потребность в оборудовании
- Система ищет свободные аудитории с учётом расписания, переносов и бронирований на конкретную дату
- Подтверждение через модальное окно с полной информацией об аудитории и мероприятии
- При отсутствии аудиторий — предложение онлайн-формата

### Расписание

Визуализация расписания с наложением переносов и бронирований:

- Выбор даты и корпуса
- Таблица: аудитории × временные слоты
- Цветовая маркировка: обычные занятия, перенесённые, забронированные

### Статистика

Аналитика для принятия решений по управлению фондом:

- **Сводка по фонду** — аудиторий, корпусов, занятий, групп, общая вместимость
- **Компьютерные классы** — использование по назначению vs трата дефицитного ресурса на обычные занятия, топ аудиторий с «пустыми» ПК-слотами
- **Загрузка по вместимости** — слоты занято/доступно с учётом потоковых лекций, переполнение категорий без подходящих аудиторий
- **Аудитории-получатели переносов** — «рабочие лошадки», закрытие которых критично для системы

Все выводы генерируются автоматически из текущих данных и меняются при изменении расписания/переносов.

### Управление

Просмотр и удаление переносов и бронирований:

- Таблица всех переносов с фильтрацией по дате
- Таблица всех бронирований с фильтрацией
- Удаление отдельных записей

### База данных

SQLite, 7 таблиц:

| Таблица | Описание |
|---|---|
| `groups` | Учебные группы (название, количество студентов) |
| `rooms` | Аудитории (корпус, этаж, вместимость, проектор, ПК) |
| `lessons` | Дисциплины (название, тип, преподаватель, оборудование) |
| `schedule` | Расписание (связь занятия, группы, аудитории, день, время, неделя) |
| `transfers` | Переносы аудиторий (schedule_id, старая/новая аудитория, дата, причина) |
| `event_bookings` | Бронирования мероприятий (аудитория, дата, время, название) |

### Алгоритм массового переноса

1. Сбор затронутых занятий из `schedule` по фильтру инцидента
2. Группировка потоковых лекций по `lesson_id` (одна аудитория на все группы)
3. Поиск свободных аудиторий через `get_free_rooms()` с учётом:
   - базового расписания
   - уже существующих переносов (IN занимает, OUT освобождает)
   - бронирований мероприятий
4. Расчёт матрицы штрафов (вместимость, оборудование, корпус, этаж)
5. Венгерский алгоритм (`scipy.optimize.linear_sum_assignment`) — оптимальное назначение
6. Сохранение результатов в `transfers` с `booking_date` на каждую дату периода

## Запуск

### Требования

- Python 3.12+
- Windows (для bat-файлов) или ручной запуск на любой ОС

### Быстрый старт (Windows)

Двойной клик на `run.bat`. При первом запуске:

1. Создаётся виртуальное окружение `.venv`
2. Устанавливаются зависимости из `requirements.txt`
3. БД скачивается из API `schedule.misis.club` (~2-3 мин)
4. Запускается Streamlit на `http://localhost:8502`

При повторных запусках предлагается выбор: использовать существующую БД или обновить из API.

### Ручной запуск (любая ОС)

```bash
python -m venv .venv
source .venv/bin/activate        # Linux/Mac
.venv\Scripts\activate           # Windows

pip install -r requirements.txt
python pipeline/build_db.py      # только при первом запуске
streamlit run src/app.py --server.port 8502
```

### Обновление расписания

```bash
python pipeline/build_db.py      # пересоздаёт БД из API
```

Внимание: переносы и бронирования при этом удаляются.
